-Folder Structure- –docs– Holds the documentation generated by Doxygen, although the Doxyfile is in the root directory –games– Holds the Game class and everything that inherits from it –ui– This folder holds all of the UI elements, by reimplementing everything in this folder you would redo the entire user interface. –ai– This folder holds the code for controlling computer players. –gamestate– This folder holds the code for representing Game States, and the rules of games –debug– This folder contains code for debugging, and nothing in this folder should be included in release code
